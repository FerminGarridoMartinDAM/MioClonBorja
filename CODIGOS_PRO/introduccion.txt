GIT -> Herramiento que permite controlar las versiones (estados) de los ficheros de una carpeta local y su asociacion con una carpeta red (repositorio)	-> GIT BASH

REPOSITORIO -> ESPACIO en el servidor de red (GITHUB) donde podremos subir los codigos. Para poder publicar en el repo necesito una rama

RAMA -> "carril" del respositorio donde puedo subir el codigo. Podemos tener tantas como necesitemos.

COMMIT -> acción que garantiza que los cambios que se realizan estan OK

PUSH -> acción que sube al repositorio red todos los cambios que previamente han sido commiteados

PULL -> acción que descarga los cambios que hay entre el repositorio de red y tu maquina

MERGE -> acción que junta varias ramas en una sola
